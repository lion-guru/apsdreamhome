Google OAuth क्रेडेंशियल्स सेट करने के लिए निर्देश:

1. Google Cloud Console पर जाएं:
   - https://console.cloud.google.com पर जाएं
   - अपने Google अकाउंट से लॉगिन करें

2. प्रोजेक्ट बनाएं:
   - ऊपर बाईं ओर प्रोजेक्ट ड्रॉपडाउन पर क्लिक करें
   - "New Project" पर क्लिक करें
   - एक प्रोजेक्ट नाम दें (जैसे "My APS Site")
   - "Create" पर क्लिक करें

3. OAuth API को एनेबल करें:
   - बाईं साइडबार में "APIs & Services" पर जाएं
   - "Enable APIs and Services" पर क्लिक करें
   - सर्च बार में "Google OAuth" टाइप करें
   - "Google OAuth2 API" को चुनें और एनेबल करें

4. OAuth कंसेंट स्क्रीन सेट करें:
   - बाईं साइडबार में "OAuth consent screen" पर जाएं
   - User Type चुनें (टेस्टिंग के लिए "External" चुनें)
   - "Create" पर क्लिक करें
   - App name, user support email, और developer contact information भरें
   - "Save and Continue" पर क्लिक करें

5. क्रेडेंशियल्स बनाएं:
   - बाईं साइडबार में "Credentials" पर जाएं
   - "Create Credentials" पर क्लिक करें और "OAuth client ID" चुनें
   - Application type में "Web application" चुनें
   - एक नाम दें (जैसे "APS Site Web Client")

6. Redirect URIs सेट करें:
   - "Authorized redirect URIs" में निम्नलिखित URL जोड़ें:
     लोकल डेवलपमेंट के लिए: http://localhost/march2025apssite/google_callback.php
     प्रोडक्शन के लिए: https://your-domain.com/march2025apssite/google_callback.php

7. क्रेडेंशियल्स सेव करें:
   - "Create" पर क्लिक करें
   - Client ID और Client Secret को सेव कर लें

8. क्रेडेंशियल्स को कॉन्फ़िगर करें:
   - google_oauth_config.php फ़ाइल में जाएं
   - GOOGLE_CLIENT_ID और GOOGLE_CLIENT_SECRET को अपडेट करें
   - डेवलपमेंट और प्रोडक्शन क्रेडेंशियल्स अलग-अलग सेट करें

ध्यान दें:
- क्रेडेंशियल्स को सुरक्षित रखें और किसी के साथ शेयर न करें
- टेस्टिंग के लिए अपना Google अकाउंट टेस्ट यूजर्स में जोड़ें
- प्रोडक्शन में जाने से पहले OAuth कंसेंट स्क्रीन को वेरिफाई करवाएं
