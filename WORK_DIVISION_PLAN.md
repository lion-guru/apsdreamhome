## 🔄 Real-Time Coordination Rules

### Communication Protocol
1. **Task Updates**: Update current task in collaboration dashboard before starting work
2. **File Conflicts**: Communicate before working on shared files
3. **Code Reviews**: Request review for major changes
4. **Sync Points**: Coordinate Git pushes and merges

### File Ownership Guidelines
- **Exclusive Files**: Work independently on assigned files
- **Shared Files**: Communicate before modifications
- **Core Files**: Require both users' approval for changes

### Conflict Resolution
1. **Identify Conflict**: Check collaboration dashboard for overlapping tasks
2. **Communicate**: Discuss work division and reassign if needed
3. **Coordinate**: Use Git branches for parallel development
4. **Merge**: Review and merge changes collaboratively

## 🚨 Emergency Procedures

### Git Conflict Resolution
1. **Stop Work**: Cease modifications immediately
2. **Communicate**: Notify other user of conflict
3. **Stash Changes**: Temporarily save local work
4. **Pull Changes**: Get latest remote version
5. **Resolve**: Manually merge conflicting changes
6. **Test**: Verify functionality after merge
7. **Commit**: Push resolved changes

### System Failure Recovery
1. **Assess Damage**: Check what functionality is affected
2. **Backup Data**: Ensure no data loss
3. **Coordinate**: Decide recovery approach together
4. **Restore**: Implement backup recovery plan
5. **Test**: Verify system functionality
6. **Document**: Record incident and resolution
