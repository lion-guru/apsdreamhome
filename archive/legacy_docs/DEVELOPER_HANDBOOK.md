# डेवलपर हैंडबुक

यह दस्तावेज़ APS Dream Home प्रोजेक्ट के लिए विकास दिशानिर्देश, कोडिंग मानक और सर्वोत्तम अभ्यास प्रदान करता है।

## 1. कोडिंग मानक

### PHP
- PSR-12 कोडिंग शैली का पालन करें।
- सभी फ़ाइलों में `<?php` टैग का उपयोग करें।
- क्लास, मेथड और वेरिएबल के लिए स्पष्ट और वर्णनात्मक नामकरण परंपराओं का उपयोग करें।
- प्रत्येक फ़ंक्शन और क्लास के लिए PHPDoc टिप्पणियाँ शामिल करें।

### JavaScript
- ES6+ सिंटैक्स का उपयोग करें।
- Airbnb JavaScript स्टाइल गाइड का पालन करें।
- सभी फ़ंक्शंस और महत्वपूर्ण कोड ब्लॉक के लिए टिप्पणियाँ शामिल करें।

### CSS
- BEM (ब्लॉक, एलिमेंट, मॉडिफायर) नामकरण परंपरा का उपयोग करें।
- नेस्टिंग से बचें या इसे न्यूनतम रखें।
- सुसंगत इंडेंटेशन और स्वरूपण का उपयोग करें।

## 2. विकास वर्कफ़्लो

1. **फीचर ब्रांच बनाएँ:** प्रत्येक नई सुविधा या बग फिक्स के लिए `main` ब्रांच से एक नई ब्रांच बनाएँ। नामकरण परंपरा: `feature/your-feature-name` या `bugfix/your-bug-fix-name`।
2. **कोड लिखें:** अपनी सुविधा या फिक्स को लागू करें।
3. **परीक्षण करें:** सुनिश्चित करें कि आपके परिवर्तन मौजूदा परीक्षणों को नहीं तोड़ते हैं और नई कार्यक्षमता के लिए नए परीक्षण लिखें।
4. **कमिट करें:** स्पष्ट और संक्षिप्त कमिट संदेशों का उपयोग करें।
5. **पुल रिक्वेस्ट खोलें:** `main` ब्रांच में मर्ज करने के लिए एक पुल रिक्वेस्ट खोलें।
6. **कोड समीक्षा:** सहकर्मी समीक्षा के लिए तैयार रहें और प्रतिक्रिया के आधार पर आवश्यक परिवर्तन करें।

## 3. डिबगिंग

- **PHP:** `error_log()` का उपयोग करें या Xdebug जैसे डिबगर का उपयोग करें।
- **JavaScript:** `console.log()` का उपयोग करें या ब्राउज़र डेवलपर टूल का उपयोग करें।
- **MySQL:** धीमी क्वेरी की पहचान करने के लिए MySQL लॉग की जाँच करें।

## 4. डेटाबेस माइग्रेशन

- डेटाबेस स्कीमा परिवर्तनों के लिए `database/migrations` फ़ोल्डर में SQL फ़ाइलों का उपयोग करें।
- प्रत्येक माइग्रेशन फ़ाइल में स्पष्ट रूप से संस्करण और परिवर्तन का वर्णन करें।

## 5. सुरक्षा

- हमेशा इनपुट डेटा को सैनिटाइज करें और वैलिडेट करें।
- SQL इंजेक्शन और XSS हमलों को रोकने के लिए तैयार किए गए स्टेटमेंट का उपयोग करें।
- संवेदनशील जानकारी को सीधे कोड में स्टोर करने से बचें।
- पासवर्ड के लिए मजबूत हैशिंग एल्गोरिदम (जैसे Bcrypt) का उपयोग करें।

## 6. प्रदर्शन

- डेटाबेस क्वेरी को अनुकूलित करें।
- अनावश्यक HTTP अनुरोधों को कम करें।
- कैशिंग तंत्र का उपयोग करें जहाँ उपयुक्त हो।

## 7. पर्यावरण सेटअप

- `includes/config/.env` फ़ाइल में पर्यावरण चर कॉन्फ़िगर करें।
- विकास और उत्पादन के लिए अलग-अलग कॉन्फ़िगरेशन का उपयोग करें।
