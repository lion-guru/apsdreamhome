# फाइल रिऑर्गनाइजेशन के बाद अपडेट गाइड

## परिचय

इस दस्तावेज़ में फाइल रिऑर्गनाइजेशन के बाद आवश्यक अपडेट्स के बारे में जानकारी दी गई है। रिऑर्गनाइजेशन प्रक्रिया के दौरान, फाइलें निम्नलिखित तरीके से पुनर्व्यवस्थित की गई हैं:

1. CSS फाइलें: `css/` से `assets/css/` में मूव की गईं
2. JS फाइलें: `js/` से `assets/js/` में मूव की गईं
3. इंक्लूड फाइलें: `include/` से `includes/` में मूव की गईं और सबफोल्डर्स में वर्गीकृत की गईं:
   - `includes/config/`: कॉन्फिगरेशन फाइल्स
   - `includes/classes/`: क्लास फाइल्स
   - `includes/functions/`: फंक्शन फाइल्स
   - `includes/templates/`: टेम्पलेट फाइल्स (हेडर, फुटर, नेविगेशन)
4. डेटाबेस फाइलें: `DATABASE FILE/` से `database/` में मूव की गईं

## अपडेट प्रक्रिया

रिऑर्गनाइजेशन के बाद, आपको निम्नलिखित स्क्रिप्ट्स चलाने की आवश्यकता है:

### 1. पाथ अपडेट स्क्रिप्ट

यह स्क्रिप्ट सभी PHP फाइल्स में इंक्लूड पाथ और एसेट पाथ (CSS, JS, इमेज) को अपडेट करती है:

```
php update_paths.php
```

यह स्क्रिप्ट निम्नलिखित परिवर्तन करेगी:

- `include("include/header.php")` → `include("includes/templates/header.php")`
- `href="css/style.css"` → `href="assets/css/style.css"`
- `src="js/script.js"` → `src="assets/js/script.js"`
- `include("functions.php")` → `include("includes/functions/functions.php")`
- `include("config.php")` → `include("includes/config/config.php")`

### 2. डुप्लिकेट फाइल हैंडलर स्क्रिप्ट

यह स्क्रिप्ट डुप्लिकेट फाइल्स को हैंडल करती है, जैसे `updated-*.php`, `*_backup.php`, और `*.php.bak` फाइल्स:

```
php handle_duplicates.php
```

यह स्क्रिप्ट निम्नलिखित कार्य करेगी:

- अपडेटेड फाइल्स (`updated-*.php`) को ओरिजिनल फाइल्स में मर्ज करेगी
- डुप्लिकेट फाइल्स को `backup_duplicates/` फोल्डर में मूव करेगी
- फाइल्स की तुलना करके नई फाइल्स को रखेगी और पुरानी फाइल्स को बैकअप में मूव करेगी

## मैन्युअल अपडेट्स

स्क्रिप्ट्स के बाद भी, कुछ फाइल्स में मैन्युअल अपडेट्स की आवश्यकता हो सकती है:

1. **विशेष पाथ**: कुछ विशेष पाथ जो स्क्रिप्ट द्वारा नहीं पहचाने गए हैं, उन्हें मैन्युअली अपडेट करें
2. **डेटाबेस कनेक्शन**: सुनिश्चित करें कि डेटाबेस कनेक्शन फाइल्स सही पाथ पर पॉइंट कर रही हैं
3. **एसेट पाथ**: जटिल JavaScript या CSS फाइल्स में एसेट पाथ की जांच करें

## वेरिफिकेशन

अपडेट्स के बाद, निम्नलिखित जांच करें:

1. वेबसाइट के सभी पेज लोड हो रहे हैं और सही दिख रहे हैं
2. सभी CSS और JavaScript सही लोड हो रहे हैं
3. सभी फॉर्म और फंक्शनैलिटी काम कर रही है
4. कोई PHP एरर या वार्निंग नहीं दिख रही है

## समस्या निवारण

अगर आपको कोई समस्या आती है:

1. लॉग फाइल्स देखें: `path_update_log.txt` और `duplicate_handler_log.txt`
2. PHP एरर लॉग देखें
3. ब्राउज़र कंसोल में JavaScript एरर देखें
4. किसी विशिष्ट फाइल में समस्या होने पर, उसके बैकअप से तुलना करें

## सुरक्षा नोट

रिऑर्गनाइजेशन और अपडेट प्रक्रिया के दौरान, सभी महत्वपूर्ण फाइल्स का बैकअप लिया गया है। अगर कोई समस्या आती है, तो आप `backup_duplicates/` फोल्डर से ओरिजिनल फाइल्स रिस्टोर कर सकते हैं।
